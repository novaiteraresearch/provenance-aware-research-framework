# Contributing Guidelines

Thank you for your interest in contributing to the Provenance-Aware Research Framework.

## How to Contribute

1. **Fork** the repository and create a branch from `main`.
2. Ensure your changes are consistent with the framework's provenance and integrity principles.
3. Write clear, descriptive commit messages.
4. Add or update documentation as needed.
5. Submit a **pull request** with a concise description of your changes.

## Standards

- All contributions must maintain schema compatibility or document breaking changes in `CHANGELOG.md`.
- Code must be accompanied by appropriate tests and provenance annotations.
- Security-relevant changes must reference the threat model in `THREAT_MODEL.md`.

## Reporting Issues

Please use GitHub Issues to report bugs, propose features, or raise methodological questions.

## Review Process

All pull requests are reviewed by the maintainer. Feedback will be provided within 7 days.
